# Resumo do Protótipo

A fase de prototipagem do projeto Meli+ teve como objetivo criar uma versão interativa e de alta fidelidade das soluções de design, a fim de validar as hipóteses e coletar feedback dos usuários antes do desenvolvimento.

## Detalhes do Protótipo:

*   **Ferramenta:** O protótipo foi desenvolvido no Figma, permitindo a criação de uma experiência clicável e com transições realistas.
*   **Fidelidade:** Foi criado um protótipo de alta fidelidade, utilizando as telas e os componentes definidos na fase de UI Design.
*   **Escopo:** O protótipo abrange os três principais fluxos de usuário:
    1.  **Descoberta e Assinatura:** Desde a landing page até a confirmação da assinatura.
    2.  **Gestão de Meios de Pagamento:** Visualização, adição e edição de métodos de pagamento.
    3.  **Dashboard do Assinante:** Visão geral dos benefícios e configurações da assinatura.

## Testes de Usabilidade:

O protótipo foi projetado para ser utilizado em testes de usabilidade com os seguintes objetivos:

*   **Validar a clareza da proposta de valor:** Verificar se os usuários entendem os benefícios do Meli+.
*   **Avaliar a facilidade do processo de assinatura:** Identificar pontos de fricção no fluxo de cadastro.
*   **Testar a usabilidade do dashboard:** Observar como os usuários interagem com as ferramentas de gerenciamento.

## Métricas de Avaliação:

Os testes de usabilidade foram planejados para avaliar as seguintes métricas:

*   **Taxa de conclusão de tarefas:** A porcentagem de usuários que conseguem completar os fluxos propostos.
*   **Tempo para completar tarefas:** O tempo médio que os usuários levam para realizar cada tarefa.
*   **Número de erros:** A quantidade de erros que os usuários cometem durante a navegação.
*   **Satisfação do usuário (SUS):** Uma medida quantitativa da percepção geral da experiência.

O protótipo é uma ferramenta fundamental para a validação do design, permitindo que a equipe colete dados reais sobre a usabilidade das soluções propostas e faça iterações antes de investir no desenvolvimento, reduzindo assim os riscos e garantindo um produto final mais alinhado às necessidades dos usuários.
